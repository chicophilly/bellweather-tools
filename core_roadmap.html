Every tool in the CORE ecosystem falls into one of five categories. These describe the fundamental nature of the tool — how people interact with it and what it does for them — which determines how it gets adopted, how it's maintained, and what kind of value it delivers. Naming convention codes: A (Analyzer), D (Dashboard), P (Planner), TR (Tracker), TK (Toolkit).

A
Analyzers

Capture structured assessments, surface patterns, and track trends over time. The user rates, reflects, or evaluates — the tool organizes results into institutional memory.

  • People Analyzer— EOS quarterly people reviews. Core values + GWC, consensus, reports.
  • Project Analyzer— Project & team health via distributed assessments. Alert thresholds.
  • Designer Timecard Analyzer— Biweekly hour classification, Ninety.io KPIs, Project Registry.
  • Client Analyzer— Client vibe metric for qualifying new clients against primary demographic.
  • Trades Analyzer— Subcontractor performance evaluation.
D
Dashboards

Present information for consumption, not editing. Lowest adoption barrier — "check it like the weather report" tools.

  • Project Progress Dashboard— Pipeline progression across IDEA, FDDA, and CA milestones.
  • Maps Dashboard— Permits & jurisdictions, job locations, PM assignments, marketing territories.
  • CORE System Dashboard— Live SharePoint list status by build phase.
  • Revenue & Pipeline Dashboard— Contract values + phase timing + resource allocation.
  • Design Capacity Dashboard— Designer workload heat map across team and timeline.
  • Production Capacity Dashboard— PM and field team workload visibility.
  • Sankey Financial Dashboards— Financial flow visualization for P&Ls, selections/allowances, cost distribution.
P
Planners

The operational nerve center. Data flows in from Trello, HubSpot, and SharePoint; humans adjust assignments, prioritize, and sequence. Most dashboards derive from Planner data.

  • Gantt Schedule— Multi-pronged planning tool feeding capacity dashboards and project progress.
TR
Trackers

Monitor where something stands in a lifecycle. The "thing" moves through stages; the tracker shows where it is, how long it's been there, and whether it's on schedule.

  • Permit Tracker— Portfolio-aware permit status. Jurisdiction checklists, time-based tracking.
  • Selections Procurement Tracker— Ordering deadlines, delivery status. Component of Selections Toolkit.
  • Absolutes Matrix— Weekly pre-construction status across permits, engineering, zoning, milestones.
TK
Toolkits

Multi-function operational workspaces where the work itself happens. The user produces a complex deliverable by pulling from multiple data sources, structuring the output, and refining it.

  • Selections Tool— Edit, Share, Track modes. Budget tracking, allowance governors, client presentation views.
  • Estimating Tool— Comp blending with age/confidence escalation. Currently V4 Excel, future web-based.
  • Levels of Development Tool— Risk-driven pre-construction scoping of development levels.
  • DA Contract Generator— Design Agreement assembly from HubSpot, Trello, past contracts, P&L data.
  • CA Contract Generator— Construction Agreement assembly. Inherits from DA, adds construction provisions.

What's Not in Play (Yet)

QuickBooks integration remains the most significant gap — financial data currently lives in its own silo, disconnected from project costing, revenue dashboards, and Sankey visualizations. BuilderTrend's API is a "walled garden" with limited read access and no meaningful write capability, which is driving the decision to build replacement tools rather than integrate. Both represent downstream dependencies: the Revenue & Pipeline Dashboard and Sankey Financial Dashboards will eventually need QuickBooks data to reach full potential, and any tool currently feeding into or pulling from BuilderTrend will need a migration path as custom replacements come online.
